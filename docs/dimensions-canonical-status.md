# Dimensions & Geometry Canonicalisation — Conversation Review

Date: 2026-03-27

## What was concluded (and implemented)

### 1) Source-of-truth hierarchy
- **Conservatory geometry**: BAC scans are definitive.
- **Loft geometry**: Sunlight set (`IMG_0829` + `SUNLIGHT LOFTS LIMITED part*`) is primary.
- **Neighbour/BCA**: secondary cross-check only.
- **XLSX**: cross-reference source, and override source where explicitly changed after drawings.

Implemented in:
- `model/data/inventory/house_inventory.json`
- `model/extract_house_inventory.py`

### 2) Room-name mapping from drawings to house model
Confirmed and encoded:
- Ground floor: `lounge -> leather`, `living room -> front`
- First floor: smallest bedroom = `office`, bay bedroom = `jackcarol`, other bedroom = `sterling`
- Loft: `bedroom 1 -> aldora`, `bedroom 2 -> elvina`
- Elvina door orientation + included area correction captured.

Implemented in:
- `house_inventory.json` (`room_alias_map`, `user_corrections`)

### 3) Radiator table policy
Confirmed and encoded:
- `radiator_inventory_current.csv` = canonical/current/used
- `radiator_inventory_change.csv` = possible future scenario only
- `radiator_inventory_extras.csv` = not used
- Width/Height values in current/change are correct as-entered.

Implemented in:
- `house_inventory.json` (`radiator_table_policy`, `radiator_implementation_status`)

### 4) Scan + XLSX dimensional outputs
Implemented machine-usable artifacts:
- `scan_dimension_points.csv`
- `room_dimension_summary.csv`
- `dimension_cross_reference.csv`
- `canonical_dimensions.csv`

Generated by:
- `model/extract_house_inventory.py`

### 5) Changed-window overrides
Confirmed:
- **Aldora window area** and **Shower window area** should use XLSX values, not drawing schedule.

Implemented in:
- `dimension_cross_reference.csv` (`status = xlsx_override`)
- `house_inventory.json` (`user_corrections`)

### 6) “No magic geometry numbers in source” for thermal models
Implemented:
- Canonical geometry file: `data/canonical/thermal_geometry.json`
- Python thermal model reads this file.
- Rust thermal model reads this file.
- Audit script verifies wiring + key dimensions.

Implemented in:
- `model/house.py`
- `src/thermal.rs`
- `model/audit_model_dimensions.py`
